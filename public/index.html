El Cerebro (Lógica C#)

La interactividad real reside en el código subyacente (*code-behind*). Aquí se implementa el patrón de acceso a datos utilizando Entity Framework Core de forma asíncrona. Explore los diferentes componentes clave del sistema utilizando las pestañas inferiores para revisar el código de inyección de dependencias, la validación y la carga de datos. 

using System;
using System.Threading.Tasks;
using System.Windows.Forms;
using AdventureAdmin.Data.Context;
using AdventureAdmin.Data.Models;
using Microsoft.EntityFrameworkCore;

namespace AdventureAdmin.Ui
{
    public partial class DepartmentForm : Form
    {
        // Inyección directa del contexto de base de datos
        private readonly AdventureWorksContext _context;

        public DepartmentForm(AdventureWorksContext context)
        {
            InitializeComponent();
            _context = context;
        }

        private async void btnGuardar_Click(object sender, EventArgs e)
        {
            if (!ValidateForm()) return;

            try
            {
                btnGuardar.Enabled = false;

                // Ruta absoluta para evitar colisiones de nombres (CS0103)
                var department = new AdventureAdmin.Data.Models.Department
                {
                    Name = txtName.Text.Trim(),
                    GroupName = txtGroupName.Text.Trim(),
                    ModifiedDate = DateTime.Now
                };

                _context.Departments.Add(department);
                await _context.SaveChangesAsync();

                MessageBox.Show("Éxito");
                this.DialogResult = DialogResult.OK;
                this.Close();
            }
            catch (Exception ex) { /* Manejo de errores */ }
            finally { btnGuardar.Enabled = true; }
        }

        private bool ValidateForm()
        {
            errorProvider1.Clear();
            bool valid = true;
            // Lógica de validación omitida por brevedad
            return valid;
        }
    }
}
using System;
using System.Threading.Tasks;
using System.Windows.Forms;
using Microsoft.EntityFrameworkCore;
using Microsoft.Extensions.DependencyInjection;

namespace AdventureAdmin.Ui
{
    public partial class DepartmentList : Form
    {
        private readonly AdventureWorksContext _context;

        public DepartmentList(AdventureWorksContext context)
        {
            InitializeComponent();
            _context = context;
        }

        private void DepartmentList_Load(object sender, EventArgs e)
        {
            // Llamada asíncrona segura sin esperar el resultado en el Load
            _ = LoadDataAsync();
        }

        private async Task LoadDataAsync()
        {
            try
            {
                var departamentos = await _context.Departments.ToListAsync();
                dgvDepartments.DataSource = departamentos;
            }
            catch (Exception ex) { /* ... */ }
        }

        private void btnNuevo_Click(object sender, EventArgs e)
        {
            // Obtener el formulario del contenedor de dependencias
            var form = Program.ServiceProvider.GetRequiredService<DepartmentForm>();

            if (form.ShowDialog() == DialogResult.OK)
            {
                _ = LoadDataAsync(); // Recargar tras guardar
            }
        }
    }
}
// Dentro de la configuración de servicios en Program.cs

static void ConfigureServices(ServiceCollection services)
{
    // ... otras configuraciones ...

    services.AddTransient<AdventureAdmin.Ui.Department.DepartmentList>();
    services.AddTransient<AdventureAdmin.Ui.Department.DepartmentForm>();

    // Esto permite que el sistema "inyecte" automáticamente
    // las dependencias (como el Contexto de BD) al crear estos formularios.
}

El Cable Invisible (DB Connection)

Para que la aplicación funcione, debe comunicarse correctamente con SQL Server. En Entity Framework Core, esto se define mediante la cadena de conexión (*Connection String*). Esta sección detalla las variaciones de conexión y cómo resolver problemas de localización del servidor, una fase crítica de la arquitectura.

🔌

Connection String (Hardcoded)

En el archivo `AdventureWorksContext.cs` (Línea 198), la conexión se define explícitamente, anulando el `App.config`. El uso de dobles barras invertidas es obligatorio en C#.

optionsBuilder.UseSqlServer("Server=JUNIOR\\SQLEXPRESS;Database=AdventureWorksDb;Trusted_Connection=True;");

🏠 Instancia Local por Defecto

Si la cadena dice `Server=.;` el punto indica que se busque la instalación principal de SQL Server en la máquina local. Falla si usa SQL Express.

Server=.;

💻 Instancia Nombrada (Express)

La mayoría de entornos educativos usan SQL Express. Se debe especificar el nombre de la máquina o `localhost` seguido del nombre de instancia.

Server=localhost\SQLEXPRESS;
